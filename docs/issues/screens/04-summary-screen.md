# Summary Screen UI/UX Issues

Status: Open

## Scope
- Route: `summary`
- Primary file: `app/src/main/java/com/zeynbakers/order_management_system/order/ui/SummaryScreen.kt`

## Owner goals
- Understand production needs by date range.
- Share chef prep output quickly.
- Audit totals and order notes confidently.

## Open issues

### P1
- Day/week/month mode controls are useful but visually dense.
- Products, daily breakdown, and order list can feel redundant in one long stream.
- Copy actions exist but need stronger visual priority when actionable.

### P2
- Unparsed lines are informative but can overpower core output.
- No saved presets for common reporting ranges.

## Acceptance criteria
- Range selection is obvious and always in view.
- Chef prep output is copyable in one dominant action.
- Daily and order-level sections are clearly separated and collapsible.
